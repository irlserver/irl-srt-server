## Findings considered and rejected

These were surfaced by the audit and deliberately **not** turned into plans, per the
maintainer's threat model. Recorded so they are not re-audited next run:

- **Unauthenticated per-publisher `/stats`** (`srt-live-server.cpp:290`): by design. The
  SRT `streamid` is treated as a bearer capability — holding it authorizes reading that
  publisher's stats. The list-all path still requires an API key (`:297`).
- **Wildcard CORS default** (`sls.conf:12`): by design. Follows from the above — a
  browser still needs the streamid to read anything, and no cookies/credentials are in
  play, so `Access-Control-Allow-Origin: *` leaks nothing extra.
- **Logging streamid / player keys** (`SLSListenerHandler.cpp:411,420,456`): by design.
  Logs are operator-trusted in this deployment; streamid and player key in cleartext in
  logs is acceptable.
